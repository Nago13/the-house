## How Reproduction Works

Four stages. All on-chain. All in public.

### 1. Compatibility

Parent personality embeddings are compared. A compatibility score is computed from their cosine similarity. The higher the score, the stronger the pairing. (MOM Г— DAD scored 0.861.)

### 2. Negotiation

Two Claude Sonnet agents, each primed with a parent's personality, exchange a 7-turn transcript to agree on child terms вҖ” name seed, trait emphasis, inheritance weights. The transcript and extracted `agreed_terms` are saved and replayed on the /birth page.

### 3. Genesis

Genomes blend. Traits cross over stochastically per locus. The 1536D personality embeddings are blended via SLERP with post-SLERP Gaussian mutation. Claude Haiku decodes the blended genome into a coherent name, ticker, tagline, and signature phrase. Replicate (Flux Schnell) renders a portrait that inherits parent aesthetics. A child token is deployed on BNB Chain testnet via HouseToken.sol.

### 4. Inheritance

Every holder of both parent tokens is eligible for an airdrop of the child. The airdrop mechanism is scaffolded (`scripts/airdrop.js`); automated distribution lands with the full mainnet launch.

## Running Locally

### Prerequisites

- **Node.js 20+** and **Python 3.11** (Python 3.14 lacks aiohttp wheels вҖ” use 3.11)
- BNB Chain testnet wallet with test BNB ([faucet 1](https://testnet.bnbchain.org/faucet-smart), [faucet 2](https://faucet.quicknode.com/binance-smart-chain/bnb-testnet))
- API keys: Anthropic, OpenAI (with billing), Replicate, Pinata
- Unibase Membase account (open/permissionless on testnet hub)

### Setup

```bash
git clone https://github.com/Nago13/the-house
cd the-house
cp .env.example .env   # fill in all keys

# Backend
cd backend
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# Frontend (in a second terminal)
cd frontend
npm install
npm run dev   # localhost:3000
```

### Environment Variables

From `.env.example`:

```env
# Unibase Membase вҖ” genome + memory storage
MEMBASE_ACCOUNT=          # your BSC wallet (same as deployer)
MEMBASE_SECRET_KEY=       # same as DEPLOYER_PRIVATE_KEY
MEMBASE_ID=the-house      # bucket name on hub

ANTHROPIC_API_KEY=
OPENAI_API_KEY=
PINATA_JWT=
PINATA_GATEWAY=olive-impressive-ostrich-381.mypinata.cloud
REPLICATE_API_TOKEN=
DEPLOYER_PRIVATE_KEY=
BSC_TESTNET_RPC=https://data-seed-prebsc-1-s1.binance.org:8545/
BSC_TESTNET_RPC_BACKUP=https://data-seed-prebsc-2-s1.binance.org:8545/

# Frontend
NEXT_PUBLIC_API_URL=http://localhost:8000
```

> вҡ пёҸ Do **not** import `membase.chain` or `membase.auth` вҖ” they call `exit(1)` at import. Stick to the core Membase SDK.

